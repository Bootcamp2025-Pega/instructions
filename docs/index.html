SDLC

  • Software Development Life Cycle (SDLC) is a process used by the software industry to design, develop, and test high-quality software.
  • SDLC aims to produce high-quality software that meets or exceeds customer expectations, reaches completion within times and cost estimates.
  • SDLC is a series of steps that provide a framework for developing software and managing it through its entire lifecycle.
  • SDLC is a process that defines the various stages involved in the development of software for delivering a high-quality product.
  • In simple words - we make developers job easier

Git flow

Source: https://nvie.com/posts/a-successful-git-branching-model/

  • Git flow is a branching model that helps developers keep track of features, releases, and hotfixes.
  • The main branches are master and develop. Feature branches are created from develop and merged back into develop.
  • When a release is ready, a release branch is created from develop. Once the release is ready, it is merged into master and develop.
  • Hotfixes are created from master and merged back into master and develop.
  • This model helps to keep the codebase clean and organized.
  • It is important to follow the Git flow model to ensure that the codebase is stable and that changes are properly tracked and managed.
  • Git flow is a widely used branching model and is recommended for most projects.
  • Our project is smaller, so we will use simpler flow.

Pipelines - Github actions

  • GitHub Actions is a CI/CD tool that allows you to automate your workflow.
  • You can create custom workflows that run on GitHub events like push, pull request, issue, etc.
  • You can use GitHub Actions to build, test, and deploy your code.
  • GitHub Actions is free for public repositories and offers free minutes for private repositories.
  • GitHub Actions is easy to set up and use, and it integrates well with GitHub.
  • You can use GitHub Actions to automate your CI/CD pipeline and save time and effort.
  • Github actions are defined in .github/workflows directory.
  • Workflow is defined in YAML format. It consists of jobs, steps, and actions.

Static code analyses

  • Static code analysis is a method of debugging by examining source code before a program is run.
  • It's done by analyzing a set of code against a set (or multiple sets) of coding rules.
  • Static code analysis can be done by tools like SonarQube, CodeClimate, or ESLint.
  • Static code analysis can help identify potential security vulnerabilities, code smells, and bugs.
  • Static code analysis can be integrated into your CI/CD pipeline to automate the process.
  • Static code analysis can help improve code quality and reduce technical debt.

Test coverage

  • Measures how much of your code is executed by tests
  • Helps to identify untested code
  • Question: Is it good to require 100% test coverage?

Static Application Security Testing (SAST)

  • Static Application Security Testing (SAST) is a type of security testing that is performed without executing the code.
  • SAST tools analyze the source code or compiled code to identify security vulnerabilities.
  • SAST tools can help identify security vulnerabilities such as SQL injection, cross-site scripting, and buffer overflows, hardcoded secrets, insecure dependencies, unvalidated inputs.
  • SAST tools can be integrated into your CI/CD pipeline to automate the process.
  • SAST tools can help improve the security of your code and reduce the risk of security breaches.
